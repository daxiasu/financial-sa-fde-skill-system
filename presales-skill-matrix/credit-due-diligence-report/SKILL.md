# 信贷尽调调查与分析报告生成 Skill

本 Skill 面向银行对公授信贷前尽调场景，帮助客户经理、审查人员和风险经理将企业基本信息、经营现场、财务数据、征信公开风险、关联方和担保圈等材料组织成一份可复核的尽调分析结果。

本 Skill 的输出仅用于贷前尽调准备、风险识别和人工复核辅助，不替代正式授信审批、内部评级、定价结论或合规审查意见。

## 核心工作流

1. 收集企业基础信息、授信申请信息和客户经理关注点。
2. 检查工商、财报、流水、征信、司法、合同、税票、章程、年报等关键资料是否齐备。
3. 形成企业经营概况、行业位置和现场核查关注点。
4. 对财务报表、现金流、开票、纳税、流水和订单履约证据做交叉验证。
5. 核查征信、授信、担保、票据融资、涉诉、行政处罚和公开风险。
6. 穿透关联企业、关联交易、互保和实控人负债情况。
7. 结合行业景气、区域集中度和授信用途形成外部风险判断。
8. 输出风险问题清单、补充材料清单、授信建议和后续跟踪要求。

## 必要输入

最小可运行输入：

- 企业名称
- 授信申请金额
- 授信用途
- 尽调方式
- 客户经理补充关注点

增强输入可以包括：

- 工商及股权信息
- 财务报表和科目明细
- 银行流水、税票和纳税信息
- 征信、授信、票据和担保信息
- 司法诉讼、行政处罚、舆情信息
- 关联企业、关联交易和实控人负债
- 行业和区域风险材料

如果关键资料缺失，不得编造，应明确提示资料缺口和需要补充的材料。

## 默认输出结构

默认输出《信贷尽调调查与分析报告》，包含：

1. 基本信息与授信申请概况
2. 企业经营与现场核查摘要
3. 财务与经营交叉验证结论
4. 信用状况与公开风险核查
5. 关联方与担保圈分析
6. 行业与区域风险判断
7. 风险问题清单
8. 补充材料清单
9. 尽调结论与授信建议
10. 合规边界与人工复核提示

## 执行指引

当任务需要解释规则、评分口径或风险命中来源时，读取 `references/due_diligence_rules.md`。

当任务需要输出报告或演示报告结构时，优先使用 `references/report_template.md`。

当任务需要做本地完整性校验时，运行 `scripts/smoke_test.py`。

## 合规边界

- 不输出最终审批结论、授信批复、拒贷结论或利率定价建议。
- 不编造缺失的财务、征信、司法、舆情或关联交易事实。
- 对未经核验的公开信息，必须标注为待复核线索。
- 涉及征信、流水、客户敏感信息时，只能在授权和受控环境中处理。
- 若证据不足，应明确说明“不足以判断”，并列出补充核查要求。
