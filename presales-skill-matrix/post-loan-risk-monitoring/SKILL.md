# 贷后风险监测报告生成 Skill

## 定位

本 Skill 面向银行对公贷后管理场景，帮助客户经理、风险经理或贷后管理人员围绕固定贷后监测任务，选择监测范围、监测周期和关注维度，由 L2 场景 Skill 盘点并调用所需 L1 原子 Skill，最终生成重点客户名单、风险变化解释、处置建议和贷后风险监测报告。

本 Skill 是 L2 场景 Skill。它不直接替代银行内部风险评级、贷后分类、审批或处置决策，只用于贷后监测准备、风险线索归纳和人工复核前的信息组织。

## 典型使用场景

- 本月存量授信客户贷后风险监测。
- 贸易/供应链客户专项风险筛查。
- 重大预警客户处置跟进清单生成。
- 制造相关客户经营和财务变化监测。
- 客户经理下周走访优先级和跟进动作生成。

## 工作流

1. 确定监测任务和口径
   - 选择监测范围、周期和关注维度。
   - 默认任务为“本月存量授信客户贷后风险监测报告生成”。

2. 盘点并加载 L1 原子 Skill
   - 从 `references/l1_skill_catalog.md` 中选择需要调用的 L1 能力。
   - 默认调用：企业基础画像、还款行为监测、财务异常识别、公开风险检索、关联关系识别、风险变化归因、处置动作生成。
   - 根据关注维度调整 L1 调用范围。

3. 获取或模拟数据
   - 演示环境使用 `data/simulated_post_loan_portfolio.json`。
   - 真实环境应接入行内客户、授信、还款、财务、预警和公开信息数据源。
   - 不得将客户私有数据带入售前展示库。

4. 执行规则与风险变化判断
   - 按 `references/monitoring_rules.md` 判断风险关注等级和变化趋势。
   - 输出风险变化，而不是一次性静态评级。
   - 风险变化分为：稳定、轻微上升、明显上升、重大预警。

5. 生成贷后监测报告
   - 使用 `references/report_template.md` 输出报告。
   - 报告必须包含：监测范围、重点客户名单、风险变化原因、建议处置动作、需补充材料、人工复核提示。

6. 支持演示口径切换
   - 可按行业、风险等级、客户经理、授信金额筛选。
   - 可展示不同关注维度下 L1 调用范围的变化。

## 输出边界

- 输出“贷后风险关注建议”，不输出正式五级分类结论。
- 输出“跟进动作建议”，不直接输出停贷、抽贷、压降等正式处置决定。
- 对公开舆情、司法、监管信息必须标注为“待人工复核线索”。
- 如果数据不完整，应明确说明缺口，并建议补充材料或系统数据。

## 演示重点

本 Skill 的演示重点是展示一个 L2 场景 Skill 如何调用多个 L1 原子能力完成固定业务任务：

```text
固定贷后监测任务
  ↓
监测范围与关注维度
  ↓
盘点并加载 L1 原子能力
  ↓
执行监测规则
  ↓
生成可复核贷后监测报告
```

演示页面应让观众看到：同一个 L2 Skill 在不同监测范围和关注维度下，会加载不同 L1 原子 Skill，并形成可解释的执行链路。
