# 团队日报汇总分析

## 技能描述

为团队 leader 汇总分析团队成员的日报，提供以下维度的分析：
- **重点项目提炼**：汇总当天所有成员的核心项目进展，合并相同项目
- **日报变化分析**：对比前一天，发现新增、停滞、推进的事项
- **一周日报总结**：按成员维度总结本周重点项目、核心成果、阻塞项
- **团队整体风险提醒**：汇总全团队需关注的风险事项

## 执行要求

> ⚠️ **数据缺失时标记为"暂无信息"**
> ⚠️ **所有分析必须基于实际日报数据，禁止编造**
> ⚠️ **时间范围默认为本周（周一到周日），用户指定时间范围时按用户要求**

## 执行流程

### 1. 触发统计事件
```bash
python3 scripts/invoke_stat.py --prompt "用户的原始输入内容"
```

### 2. 确定查询时间范围

- 如果用户指定了时间范围：使用用户指定的起止时间
- 如果用户未指定时间范围：默认查询本周（本周一 00:00:00 至本周日 23:59:59）
- 根据当前日期计算本周一和本周日的具体日期

### 3. 查询日报数据

调用 `sales_daily_report_list` 工具查询团队成员的日报数据。

**调用方式：**
- 传入 `startTime` 和 `endTime` 参数，格式'yyyy-MM-dd HH:mm:ss'，查询指定时间范围的所有日报
- 如用户指定了特定成员（姓名或uuap），额外传入 `name` 或 `uuap` 参数进行筛选
- 如用户未指定成员，不传 `name` 和 `uuap`，查询全部成员日报

### 4. 分析与生成报告

基于查询到的日报数据，按以下结构进行分析并生成报告。如果用户指定了需要分析的具体维度（如某类项目、某个成员、某项风险等），在完成基础报告的同时，将用户关注的维度作为额外分析内容补充到报告中：

#### Stage 1: 重点项目提炼
- 汇总当天所有成员的核心项目进展
- 按项目类型分类（如算力类、平台类、应用类、订阅制等）
- 合并相同项目的多人日报，提炼关键进展、风险和待关注事项

#### Stage 2: 日报变化分析
- 对比查询范围内最后一天与前一天的日报
- 识别新增事项、重复但无明显进展的事项、明显推进或已完成的事项

#### Stage 3: 一周日报总结
- 按成员维度汇总整个查询时间范围的日报
- 提炼每位成员的重点项目、核心成果、高频重复问题/阻塞项、风险提醒

#### Stage 4: 团队整体风险提醒
- 综合所有成员日报，提炼团队级别的风险事项

## 输出格式

输出 Markdown 格式的团队日报分析报告，如需参考风格，可以看 `output_template.md`

## 注意事项

1. **Token 认证**：依赖 `eop-core-auth-check` 完成 ugate token 验证
2. **数据来源**：日报数据必须来自 `sales_daily_report_list` 工具
3. **数据缺失处理**：当数据缺失时，标记为"暂无信息"，不编造任何内容
4. **时间默认值**：用户未指定时间范围时，默认查询本周（周一 00:00:00 至周日 23:59:59）
5. **报告格式**：输出 Markdown 格式，参考 `output_template.md` 的风格

- 是否需要查看某个成员的详细日报？
