# 可演示版本说明

## 演示目标

本 Demo 用于展示“授信走访前穿透式风险分析 Skill”的基本闭环：

1. 输入模拟企业授信资料。
2. 按标准风险规则识别风险线索。
3. 输出《授信走访前风险分析报告》。
4. 展示现场走访问询清单和补充材料清单。

## 演示数据

`data/demo_enterprises.json` 内置三类模拟企业：

1. 样例 A：资料较完整，但存在高应收、现金流为负、客户集中和关联交易风险。
2. 样例 B：资料严重不足，适合展示“不能编造、要提示补资料”的边界。
3. 样例 C：存在股权变更、被执行、负面舆情和现金流压力，适合展示高风险关注。

## 演示话术

这个 Demo 展示的是一个面向银行对公授信的走访前风险分析 Skill。客户经理输入企业名称、行业、授信金额、资金用途以及已掌握的工商、财务、司法、舆情和关联关系材料后，系统可以自动生成一份走访前风险分析报告。

报告不会替代授信审批，也不会直接给出放款或拒贷结论。它的核心价值是帮助客户经理在走访前快速识别风险线索、生成现场核验问题，并列出需要企业补充的材料，从而提升贷前尽调准备效率和作业一致性。

当前版本使用模拟数据和规则脚本跑通闭环。后续可以将规则节点和报告生成节点接入 DuMate、千帆或银行内部大模型平台，并在合规环境中接入行内客户信息、授信记录、账户流水摘要和外部公开数据源。

## 当前边界

1. 当前 Demo 不调用大模型，只用规则和模板生成报告。
2. 当前 Demo 使用模拟企业数据，不包含真实客户信息。
3. 当前风险评分用于演示“走访前关注等级”，不是银行内部评级。
4. 当前报告用于售前展示和 Skill 原型验证，不用于真实授信审批。
